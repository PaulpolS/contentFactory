# วิธีทำงานหลายเครื่องโดยไม่ให้ Git พัง

เอกสารนี้ commit ไว้ใน repo จึงเดินทางไปทุกเครื่องพร้อมโค้ด อ่านได้ทุกที่

## 🔑 กฎเหล็กข้อเดียว

> **pull ก่อนเริ่ม · push เมื่อเลิก — ทุกเครื่อง ทุกครั้ง**
> อย่าเดินจากเครื่องไหนไปโดยที่ยังไม่ push

99% ของอาการ "Git มารวมแล้วพัง" มาจากการ **ลืม pull ก่อนเริ่มแก้** หรือ **ลืม push ตอนเลิก**
ทำให้สองเครื่องแก้คนละทางแล้วมาชนกันทีหลัง

## ✅ วิธีใช้งานประจำวัน (ง่ายสุด)

ใช้สคริปต์ที่เตรียมไว้ — ดับเบิลคลิกเอา ไม่ต้องพิมพ์คำสั่ง:

| เมื่อไหร่ | ทำอะไร |
|----------|--------|
| **ก่อนเริ่มทำงานในเครื่องใดก็ตาม** | ดับเบิลคลิก `Sync.command` (จะดึงของล่าสุดมาให้) |
| เปิดโปรแกรม | ดับเบิลคลิก `StartApp.command` |
| ปิดโปรแกรม | ดับเบิลคลิก `StopApp.command` |
| **ทำงานเสร็จ / ก่อนเดินจากเครื่องนี้** | ดับเบิลคลิก `Sync.command` อีกรอบ (จะเซฟ + อัปขึ้น GitHub) |

`Sync.command` ทำให้ครบ 3 ขั้นในคลิกเดียว: เซฟงานเครื่องนี้ → ดึงของใหม่ → อัปขึ้น GitHub
ถ้าชนกัน (conflict) มันจะ **หยุดและเตือน** ไม่ลบงานทิ้ง

## 🛟 ถ้าเจอ conflict

งานไม่หายครับ — `Sync.command` commit งานคุณไว้ก่อนแล้วเสมอ
ให้เปิด Claude แล้วพิมพ์ว่า **"ช่วยแก้ git conflict ให้หน่อย"** แล้วทำตามที่บอก

## 📌 ของที่ "ไม่" commit ลง Git (จงใจ — ดูใน .gitignore)

ของพวกนี้สร้างใหม่ได้เองในแต่ละเครื่อง จึงไม่เก็บลง Git เพื่อกันไฟล์ชนกัน:

- `node_modules/` — `StartApp.command` ลงให้เองเมื่อ `package.json` เปลี่ยน
- ไฟล์มีเดีย / ฐานข้อมูล / ผลลัพธ์ที่ระบบสร้างขึ้น (รูป, วิดีโอ, `.db`, exports)
- ไฟล์ระบบ (`.DS_Store`, cache)

> ⚠️ ถ้าจะเพิ่ม "ความสามารถใหม่" ที่ต้องลง package ใหม่ ให้แก้ที่ `package.json`
> แล้ว commit `package.json` + `package-lock.json` ไปด้วย — เครื่องอื่นจะลงให้เองตอนเปิด `StartApp.command`

## 🍎 เรื่อง iCloud

อย่าย้ายโฟลเดอร์โปรเจกต์ไปไว้ใน **Desktop** หรือ **Documents** เพราะสองโฟลเดอร์นั้นถูก iCloud sync
(ทำให้ไฟล์ถูกดึงขึ้นคลาวด์แล้วเปิดไม่ติด) ตำแหน่งปัจจุบัน `~/โปรแกรมทำContent/` อยู่นอก iCloud แล้ว ปลอดภัยดี
